Polymarket Sports Scalp Cockpit

Canonical operating spec for all-sports and World-Cup-priority binary-market scalping: buy one side, buy the opposite side later, and lock the matched pair below $1.

Core objective

entry side + hedge side < 1.00
profit/share = 1 - entry - hedge
ROI = profit / (entry + hedge)

Required hedge price

required_hedge = 1 / (1 + target_roi) - entry_price

Example: U2.5 @ 0.48 targeting 8.7% ROI needs O2.5 at about 0.44.

Default unit

All ladders default to 400 shares unless the operator specifies another size.

400-share ladders

Execution correction

Bid/ask and visible depth matter. Last price and midpoint are not enough for a trade recommendation.

No executable depth = blocker

Non-negotiable rules

  1. Every trade is a two-leg inventory trade unless explicitly marked as final directional exposure.
  2. Always name the opposite hedge side.
  3. Always compute breakeven, 2%, 5%, 8%, 10%, and 12% hedge prices.
  4. Do not recommend an entry unless the opposite side can realistically decay or reprice.
  5. Bad events require a plan: wait, cut, average, partial hedge, or abandon. Do not blindly lock losses.
  6. Separate scalp path from gun-to-head final-score prediction.
  7. Operator screenshots override stale web data.

Primary scalp types

Under 2.5 decay scalp

Buy U2.5, hedge O2.5 later. Needs quiet first 10-15 minutes and no sticky live danger.

BTTS No decay scalp

Buy BTTS No, hedge BTTS Yes later. Often survives a 1-0 favorite lead better than U2.5.

Underdog +1.5

Buy dog +1.5, hedge favorite -1.5 later. Anti-blowout path; requires compact underdog start.

Favorite -1.5 event

Buy favorite -1.5 only when an event path exists: early goal, red card, or overwhelming pressure.

Under 3.5 safer scalp

Lower ROI, better survival after one goal.

Under 1.5 fragile scalp

Usually avoid. One early goal destroys the path.

Dashboard requirements

  • Auto-refresh constantly; no manual-refresh dependency.
  • World Cup priority sorting across all active sports events.
  • Read-only status, refresh interval, scan duration, last error, last-good snapshot.
  • Market board with executable bid/ask/depth and normalized pairs.
  • Candidate rows with entry, opposite hedge, hedge target grid, 400-share ladder, lockable depth, and failure plan.
  • Visible blockers: no token, no book, stale book, thin depth, no timing, unclassified rules, no realistic hedge path.

Position screenshot math

total_cost = cost_side_A + cost_side_B

if side A wins: profit = shares_A - total_cost
if side B wins: profit = shares_B - total_cost

if same shares both sides:
  guaranteed_profit = shares - total_cost
else:
  minimum_guaranteed_profit = min(shares_A, shares_B) - total_cost

Safety boundary

No execution path. This cockpit never reads secrets, wallet keys, API keys, or live-control files. It never submits, cancels, arms, approves, or simulates live orders. Existing Rust NativePM/polytrade services are donor evidence only; any future live path must be Oloraculo-owned, disabled by default, and separately gated.
